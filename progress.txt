## Codebase Patterns
- Dotfiles OpenCode assets are installed by `make install` from `ai/opencode/` into `$HOME/.config/opencode/`.
- OpenCode commands live under `ai/opencode/commands/` and are copied to `$HOME/.config/opencode/commands/`.
- OpenCode skills live under `ai/opencode/skills/` and are copied to `$HOME/.config/opencode/skills/`.
- Reusable standalone OpenCode prompt files can be installed directly under `$HOME/.config/opencode/`, following the existing `ai/ralph.md` pattern.
- OpenCode command files use markdown with frontmatter and can gather context through inline shell expansions using the same `$1` argument pattern as existing commands.
- Keep PR review objectives and shared schemas in `ai/opencode/pr-review.md`; keep command-specific orchestration and context-gathering in `ai/opencode/commands/review-pr.md`.
- Backup and cleanup behavior must stay documented in both README.md and AGENTS.md when `make install` or `make clean` changes.
- Keep diff line mapping and inline review payload preview rules in `ai/opencode/commands/review-pr.md` because they are command-specific orchestration.
- `/review-pr --post` must preview the exact GitHub review command or API payload and require explicit confirmation before any `gh` write command is run.
